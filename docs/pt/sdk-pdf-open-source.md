# SDK PDF Open Source

Um SDK PDF open source entrega os blocos para renderizar, buscar e anotar documentos com controle total de UX, dados e roadmap. Ele e ideal para times que precisam de um leitor confiavel sem lock-in de fornecedor.

## O que voce ganha com um SDK PDF

- Render e text layer para selecao, destaques e sumario.
- Componentes de UI como toolbar, sidebar e paginacao.
- Estado e eventos para anotacoes, busca e navegacao.
- Temas para manter o viewer alinhado ao produto.

## Quando escolher um SDK PDF open source

- Voce precisa de um leitor customizado com ownership total.
- Seu produto precisa rodar no web e no mobile com logica compartilhada.
- Quer manter busca e anotacoes dentro do seu app.

## Como o Papyrus se encaixa

Papyrus e um SDK PDF open source com core compartilhado, engines plugaveis e UI para React e React Native. Ele foca em fluxos reais de leitura: selecao, busca, destaques e anotacoes.

Pontos chave:

- Licenca MIT sem limites de uso.
- Core agnostico (PDF.js no web, engine nativa no mobile).
- Componentes prontos para leitores web e mobile.
- Suporte multi-formato (PDF, EPUB, TXT) se sua roadmap precisar.

## Checklist de avaliacao

- Licenca open source com termos claros
- Qualidade de render e text layer
- Busca, sumario e navegacao por pagina
- Anotacoes e eventos de selecao
- Paridade web e mobile
- Docs e exemplos ativos
