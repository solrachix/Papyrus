# Melhor SDK PDF Gratis 2026

Se voce procura o melhor SDK PDF gratis em 2026, foque em licenca open source, flexibilidade de engine e no fluxo completo de leitura. Este guia explica o que avaliar e onde o Papyrus se encaixa.

## O que "gratis" deve significar em 2026

- Licenca open source clara (MIT, Apache-2.0, GPL) sem limites escondidos.
- Manutencao transparente e roadmap publico.
- Sem paywall para recursos basicos como busca, selecao e sumario.

O Papyrus tem licenca MIT e e open source, o que facilita avaliar e usar em produtos comerciais.

## O que avaliar em um SDK PDF

1. **Qualidade da engine**: PDF.js, PDFium ou engines nativas com bom render e text layer.
2. **Camada de UI**: componentes para viewer, toolbar, sumario e busca.
3. **Anotacoes**: destaques, notas e estado consistente de selecao.
4. **Busca e navegacao**: busca full-text, outline e controle de pagina.
5. **Multi-formato**: PDF com suporte a EPUB e TXT quando necessario.
6. **Paridade web e mobile**: React no web e React Native no mobile.

## Categorias da shortlist

### 1) Engine apenas
Se voce so precisa renderizar, PDF.js pode bastar. Mas voce ainda precisa criar UI, estado e anotacoes.

### 2) SDKs open source modulares
Aqui entra o Papyrus: core compartilhado, engines plugaveis e UI para React e React Native. Ele foi pensado para leitores de documentos, nao apenas render.

### 3) SDKs comerciais com free tier
Alguns vendors tem tier gratuito, mas costumam limitar anotacoes ou busca avancada.

## Por que Papyrus e uma boa opcao

- SDK PDF open source com licenca MIT.
- Core agnostico que funciona com PDF.js e engines nativas.
- Componentes de UI para leitores (viewer, sidebars, topbar).
- Estado e eventos para anotacoes, busca e selecao.
- Suporte a PDF, EPUB e TXT.

## Checklist rapido

Use este checklist para comparar opcoes:

- Licenca open source com termos claros
- Suporte web e mobile
- Busca e sumario integrados
- Anotacoes e eventos de selecao
- Temas e controle de UI
- Docs e exemplos ativos
